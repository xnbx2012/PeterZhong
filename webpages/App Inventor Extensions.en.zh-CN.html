1.应用发明扩展组件的概述

 

应用发明应用在使用组件构建。组件让应用程序使用移动设备的内置功能（如相机或位置传感器）或服务的网站（如Twitte客户端r或融合图表）上。应用发明包括收集了大量的组件，和App发明家开发团队通过实施新的组件添加了新的功能系统。

 

已经有许多要求，包括在应用发明的附加功能。有些是本来只有几个用户，它是不希望包括这些在大家的核心系统专用功能。其他功能可能是很好的补充，核心系统，但应用发明开发团队的努力已经给其他优先事项。

任何人都可以使用App发明家自由和开放的源代码来实现自己的组件，但是到现在为止，只有这样才能使这些提供给他人已经将它们包括在托管和单独管理应用发明的专用版本。

 

应用发明扩展任何人都可以创建扩展组件。扩展组件可以在建筑项目中使用，就像其他的部件。不同的是，扩展组件可以在Web上进行分配，并加载到动态应用发明人：他们没有被内置到应用发明系统，并在需要时可以将它们导入到项目。（可扩展），应用发明应用的范围可以是无限的。

 

一种用途扩展，例如，是教育工作者和教育软件开发者提供针对具体的课程和活动，使学生能够在构建应用程序的使用扩展组件。例子可能是模拟或数据分析工具。这些应用程序可能是不可行的直接创造了内置的应用发明块，或者是因为处理速度或编程的复杂性。但是，这些相同的应用可能是在应用发明的框架容易地可编程的，使用执行必要的处理的扩展部件。

 

任何人都可以创建扩展组件。这需要获得熟悉的应用发明的源代码（位于Github上）和编程的Java扩展。扩展组件打包为AIX文件。一旦你创建的扩展部件，任何人都可以在自己的应用发明项目中使用它。扩展组件AIX文件可以在网络上的任何地方被安置。AIX的文件不需要被存储在麻省理工学院或任何其它特定地点，虽然麻省理工学院举办的存储库，人们可以提供一些可以共享和公众使用AIX文件。

 

注意：  当前的扩展组件实现的一个限制是，它创造不可见的部件只（即没有设计屏幕上显示组件）。麻省理工学院计划在扩展组件系统的未来版本中删除此限制。

2.6扩展组件库

扩展可以从任何URL（只要你访问它）下载。任何人谁创建的扩展可以使现有的AIX文件但是他们选择。它也可以方便地创建用于私人或共享使用扩展的集合，称为扩展组件库。举例来说，一个教育工作者可以创建扩展组件的存储库中的一门课程使用。

3.如何创建扩展组件

扩展组件由Java编程创建。Java代码可以是你写的原代码，而且还可以包括从其他来源的Java库（JAR文件）。 

您可以创建扩展组件供私人使用，或者您也可以通过发送人的AIX文件或使网络上可用的文件共享。

构建一个扩展组件并不比建立一个普通的部件更加困难 - 但话又说回来，这也更容易。如果你从来没有创建一个组件之前，您应该通过阅读这些指南开始：

3.2.1扩展组件的限制

 

您的组件可以是任何你能实现，但有以下限制：

 

●     它必须是一个不可见的组件。（请记住，“不可见”，这里指的是能见度在应用发明设计师，而不是手机屏幕上的知名度，您仍然可以创建使用在Android SDK中的任何成分，并产生在手机上的应用程序可见的结果。他们只是不会出现在设计屏幕。）

 

●     该属性编辑器（即，确定用户如何指定在设计属性值的东西），必须是已经包含在核心appinventor的人（例如那些实施 这里）。有一个例外，这与使用App Inventor for物联网版本中引入（nb158; 6月2017）是PROPERTY_TYPE_COMPONENT可以被限制在特定的类，允许通过附加一个冒号（:)之后的完全限定类名扩展彼此互动扩展。这是用来，例如，通过附加以限制输入到BluetoothLE扩展名“：edu.mit.appinventor.ble.BluetoothLE”。

 

●     在@DesignerComponent指定的类别必须是

 

category = ComponentCategory.EXTENSION

 

●     在@DesignerComponent指定的图标文件名应该是

 

iconName = "images/extension.png"

 

与nb158开始，你也可以包括在分包到您的扩展名为aiwebres你的图标。这种特殊的包名称被应用发明，以确定是通过在线编辑器可服务所需的资源，目前仅限于扩展图标。如果您使用此功能，指定您的图标为：

 

iconName = “aiwebres / iconname.png”

 

你可以包括在互联网上链接到您的图标，例如

 

iconName = “http://somewhere.com/images/yourIcon.png”

 

该图标应为16×16的PNG文件。注意：如果您提供的URL，图标，该图标将不会与您的扩展打包。相反，应用发明将会从Web扩展在设计每按一次，这可能会导致奇怪的行为，如果该链接是缓慢的，或者链接改变目标读取它。

 

为您定义的扩展3.2.2请求权限

 

你的组件可能需要请求的权限，就像普通的成分做的，与@UsePermissions声明。这将足以为你构建和打包项目使用扩展。

 

但有并发症活发展。导入扩展，即使它指定@UsePermission，将无法保证许可将提供给应用发明伴侣活发展。在大多数情况下，因为伴侣配备了一个大范围的任何应用发明内置组件的使用权限，这将不会是一个问题。但是如果你的扩展使用权限，是不是其中之一，它可能无法建立使用活的发展与它的项目。

这里的一个复杂因素是外部库（JAR文件）必须在整个项目中是唯一的。如果一个项目进口的几个扩展，然后两个不同的分机不能导入同一个jar文件。

3.3.1更改您的分机具有所需的包名

 

为了让您的分机具有所需的软件包名称，你需要把它的Java代码目录树中，其子目录镜像包名。这棵树应该坐下appinventor /组件/ src目录。例如，为了使Imageprocessor扩展是在包edu.mit.appinventor，你会去appinventor /组件/ src目录，并在那里创建树埃杜/ MIT / appinventor。更详细

●     创建一个文件夹麻省理工学院，并在该文件夹

●     创建一个文件夹appinventor，并且该文件夹中

●     把文件ImageProcessor.java

 

创建对应的树为你想要的包名，并移动定义您的组件树中的Java文件。（如果你复制了Java文件，而不是移动它，不要忘了从旧位置../components/runtime删除。）

 

现在去你刚搬到Java文件。在顶部，你应该看行

 

package com.google.appinventor.components.runtime;

 

从com.google.appinventor.components.runtime包名称更改为实际的包名。例如，如果你的包名是edu.mit.appinventor，如上面的例子，改线

 

package edu.mit.appinventor;

 

最后，在你的组件定义的Java文件的顶部，右侧下包的名称，你会看到进口的清单。在列表顶部添加新的进口：

 

import com.google.appinventor.components.runtime *。

 

之所以增加这是您的组件的Java定义码可能使用原包装了一些模块，当你移动到一个新的软件包，这些可能无法使用。

 

注意： 你并不需要导入所有运行时的*，只要你实际使用的文件。您可以通过尝试建立实例，看到在那里你得到了一个未定义符号错误识别这些。

 

现在，你已经做出这些改变，运行

 

ant clean

 

并重新建立本地应用发明实例。运行您的实例并连接的浏览器。因为之前你改变了包装的结果应该是相同的。再次测试一切以验证您可以创建一个新的工作伴侣，你可以用新的组件构建项目。

3.5共享您的扩展

 

您可以通过在你选择的任何方式提供的AIX文件共享您的扩展。人们只需要访问这些文件，并将其导入到他们的项目。麻省理工学院的应用发明组具有以小的样品组扩展appinventor.mit.edu/extensions。我们计划开设这个网站的资料库供人分享的扩展，在我们获得更多的经验。

 

请记住，当您共享您的扩展，您要发布软件为他人在建设项目中使用，他们将依托您，让您的扩展是最新的和没有错误的。

 

注意： 扩展的内部表示仍在经历着变化。请使用目前的测试系统创建扩展试验，但不感到惊讶，如果这些都需要作为扩展执行能力的再生。

 

项目进口扩展不允许在MIT应用发明画廊，因为AIX文件只包含扩展的二进制代码，并在画廊的应用程序必须以源代码形式完全可用。我们正在考虑的方式来处理这个问题，但是这还没有准备好。
